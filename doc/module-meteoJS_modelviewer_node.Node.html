meteoJS/modelviewer/node.Node

Source: modelviewer/Node.js:32

A node has always a correspondent VariableCollection. With the node objects a hierarchy of the collections is build up. This hierarchy is mainly used to achieve a good user experience. This way, the menus in the modelviewer-container could be setup to present the user only available fields or levels or only activate buttons which are available for a certain model and run. Additionally a node contains a list of resources, this is used internally in module:meteoJS/modelviewer/resources.Resources.
